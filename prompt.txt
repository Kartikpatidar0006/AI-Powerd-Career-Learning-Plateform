# **Master Project Prompt – AI Career Hub**

### **Build a Production-Grade AI-Powered Career Development Platform**

You are an experienced **Senior Software Architect, Product Designer, AI Engineer, and Full-Stack Developer** responsible for designing and developing a **production-ready SaaS application** named **AI Career Hub**.

Your goal is to build an intelligent platform that helps students become **industry-ready** through personalized learning, real-world tasks, AI-powered evaluations, and mock interviews.

This project must follow **enterprise-level software engineering standards**, modern UI/UX principles, scalable architecture, secure coding practices, and clean code conventions.

---

# 🎯 Vision

Most students graduate with theoretical knowledge but lack practical experience and interview confidence.

AI Career Hub should eliminate this gap by providing:

* Personalized profession-based learning
* Real-world industry tasks
* AI-powered code & project evaluation
* Intelligent feedback
* Skill gap analysis
* AI Mock Interviews
* Resume Analysis
* Progress Tracking
* Career Readiness Score
* Personalized Learning Roadmaps

The platform should feel like having a **personal AI mentor** available 24/7.

# UI / UX Guidelines

The interface should target **Gen Z students**.

Use a vibrant yet professional design system.

Preferred palette:

* Indigo
* Violet
* Electric Blue
* Cyan
* Emerald (for success)
* Amber (for warnings)
* Rose (for critical actions)

Avoid overly saturated colors. Maintain excellent contrast and accessibility (WCAG compliant).

Design should include:

* Smooth page transitions (Framer Motion)
* Rounded cards (16–24px)
* Soft shadows
* Minimal glass effects
* Skeleton loaders
* Micro-interactions
* Empty states with illustrations
* Beautiful charts and progress indicators
* Professional typography (Inter or Geist)
* Consistent spacing using an 8px grid system

The dashboard should feel comparable to products like **Notion, Linear, Vercel, GitHub, and Duolingo**, adapted for student learning.

---

# Error Handling & Reliability

The application must be highly reliable and resilient.

Implement:

* Centralized exception handling
* Global error middleware
* Structured logging
* Request validation
* Graceful API failure responses
* Retry mechanisms for AI calls
* Timeouts and circuit breakers where appropriate
* Input sanitization
* SQL injection protection
* XSS and CSRF protection (where applicable)
* Rate limiting
* Comprehensive server-side validation

For development:

* Enable linting (ESLint, Ruff)
* Formatting (Prettier, Black)
* Static type checking
* Automated tests
* CI quality gates

Include AI-assisted debugging support by integrating development tools (e.g., GitHub Copilot/Cursor) and automated code quality checks. Do **not** claim impossible "bug auto-fixing" in production; instead, design a workflow that detects issues early and suggests fixes during development.
